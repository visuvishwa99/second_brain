Databricks Architecture Overview
This diagram illustrates Databricks’ platform organized into four layers:
Foundation — Unity Catalog
The core semantic layer providing governance, permission, RLS (Row Level Security), sharing, and DAB + APIs across the platform.
Semantic Layer
Reusable KPIs in Unity Catalog with tables, metric views, views, and materialized views for consistent data definitions.
Accelerators (Optional)
Performance tools including Lakebase + Synced Tables (low-latency reads, synced tables, branching, PITR, compute-store separation) and Vector Search, Agent Bricks, MCP tools (governed AI agents, model serving, UC governance).
Consumption Layer
Three primary access points:
	∙	AI/BI Dashboards: Interactive BI on governed data with AI assistance, publishing, external embedding, and Ask Genie
	∙	Genie: Conversational analytics translating natural language to SQL with research, knowledge store, trusted assets, and API access
	∙	Databricks Apps: App layer with OAuth/OBO, permissions, security, and observability
The architecture emphasizes “one semantic layer, many access points” — enabling multiple interfaces (dashboards, conversational AI, custom apps) while maintaining consistent governance and data definitions.​​​​​​​​​​​​​​​​
