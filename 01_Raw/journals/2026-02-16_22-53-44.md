Yes — AWS Lambda primarily scales horizontally, not vertically.

  

  

🔹 Horizontal Scaling (What Lambda Does)

  

  

- When traffic increases, Lambda creates more function instances.
- Each request can run in its own separate execution environment.
- This is automatic — no server management needed.
- Example:  
    

- 1 request → 1 instance
- 1,000 concurrent requests → up to 1,000 instances (within concurrency limits)

-   
    

  

  

This is classic horizontal scaling (scale out).

  

  

  

  

🔹 Vertical Scaling (Limited but Configurable)

  

  

Lambda does not dynamically scale vertically at runtime, but:

  

- You choose the memory size (128 MB → 10,240 MB).
- CPU power scales proportionally with memory.
- If you increase memory, you effectively increase CPU.

  

  

So:

  

- Vertical scaling = manual configuration.
- Horizontal scaling = automatic.

  

  

  

  

  

⚡ Important Concept

  

  

Lambda is designed for:

  

“Scale out, not scale up.”

  

If you need long-running processes or massive single-instance compute, services like:

  

- Amazon EC2
- Amazon ECS
- Amazon EKS

  

  

might be better.
