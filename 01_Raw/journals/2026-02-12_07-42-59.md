How do you track DBT model logic versions?

I'm building daily append-only snapshot of a table and need to know which version of the logic produced each day’s data. The challenge: changes can happen not just in the model itself, but also in upstream int models.

Approaches I'm considering:
1) Manual var() in dbt_project.yml - simple integer, bump it when logic changes. Bleh... doesn't track upstream changes, easy to forget.
2) dbt_artifacts checksum - the package stores a SHA hash of each model’s SQL per run. Automatic but only tracks individual model, not upstream changes
3) Compile-time macro using graph.nodes - combine checksums from the model + key upstream models into one hash at compile time. Catches upstream changes automatically but produces a hash, not a readable version number.
4) dbt model version: in YAML - native feature, but static/manual - same limitation as approach 1.

Leaning toward #3 but curious - has anyone solved this differently?
