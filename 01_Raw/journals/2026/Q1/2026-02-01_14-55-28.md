## 📝 Notes

- LangChain, LangGraph, and LangSmith are a suite of tools from LangChain Inc. for building applications with large language models (LLMs).

**LangChain** is a framework for developing LLM-powered applications. It provides abstractions and components for connecting language models to data sources, APIs, and other tools. Key features include prompt templates, chains (sequences of calls), agents (LLMs that decide which actions to take), and integrations with vector databases for retrieval-augmented generation (RAG). It's available in Python and JavaScript.

**LangGraph** is a library built on top of LangChain for creating stateful, multi-actor applications with LLMs. It's designed for building complex agent workflows and multi-agent systems. The core concept is modeling your application as a graph where nodes represent computation steps (like LLM calls or tool use) and edges define the flow between them. This makes it easier to build applications with cycles, branching logic, and persistent state—things that are harder to express with simple chains.

**LangSmith** is a platform for debugging, testing, evaluating, and monitoring LLM applications. It provides observability into your chains and agents, letting you trace runs, log inputs and outputs, compare model performance, and identify issues. It's particularly useful for moving from prototyping to production, as it helps you understand why your application behaves a certain way and track performance over time.

In practice, many teams use all three together: LangChain for the core application logic, LangGraph when they need complex agent workflows, and LangSmith for development tooling and production monitoring.
