I enforce AI safety with if-statements, not prompts.

"Please don't delete important files" is a hope.
A pre-execution hook that blocks rm -rf is a guarantee.

Not philosophy. Architecture.

Claude Code, arguably the most effective AI coding agent/harness right now, follows the same principle.

The model reasons. Everything around it enforces.

Permissions? Deterministic code.
Sandboxing? OS-level isolation.
Lifecycle hooks? Scripts that intercept before the model acts.

Anthropic's own docs say it best:
"CLAUDE.md instructions are advisory. Hooks are deterministic."

Advisory = hope.
Deterministic = guarantee.

That distinction rewired how I think about building AI systems.

The model provides intelligence.
The harness provides reliability.
One is probabilistic. The other never fails.

The emerging AI Engineer role isn't about making everything run through a model.

It's about knowing where the boundary is, and enforcing it with the boring, reliable tools we've always had.

What's your line between "let the model handle it" and "just write the code"?
