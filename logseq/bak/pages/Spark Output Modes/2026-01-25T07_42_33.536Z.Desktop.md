## Spark Output Modes
- Spark Structured Streaming processes data incrementally in micro-batches. The output mode defines what data is written to the sink.
  
  ---
- ### [[OM:append]] (Default)
- If the table is not created a new table is created for the first microbatch then for the next microbatch data is appended.
- **Stateless transformation**.
- **Usecase**: Long-term storage of logs/events (collect and append without transformation).
  
  ---
- ### [[OM:complete]]
- All rows in the streaming "DataFrame/Dataset" will be written to the sink every time there are updates.
- **Usecase**: Mandatory for queries containing streaming aggregations (counts, sums, averages).
- **Materialized Views**: Ideal for dashboards requiring a full refresh.
- *Note*: Resource-intensive with large datasets as it stores the entire result in memory.
  
  ---
- ### [[OM:update]]
- Only the rows that were updated in the streaming "DataFrame/Dataset" will be written to the sink.
- **Usecase**: Efficient incremental updates using `foreachBatch`.
- Supports **Upserts** (Merge + Insert).
- Includes advanced streaming concepts like **Watermarking** and **Windowing** (Tumbling/Sliding).
