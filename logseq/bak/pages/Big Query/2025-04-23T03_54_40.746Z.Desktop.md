	- ## Architecture for GCP BIG query
		- Storage => Colossus (Google's distributed file system for data storage)
		- Compute => Dremel (Root server  [Coordinates query execution], Mixers  [Aggregate data], Leaf nodes [Read data from storage] )
		- ^^NM^^ : Dremel is like camel
		- Connecting Components =>
			- Jupiter:
			  High-speed network protocol connecting Dremel (compute) and Colossus
			  (storage)
			- Borg:
			  Orchestrator managing internal communication between components
		- ### Query Execution Flow
		  
		  1. The user submits a query
		  2. The root server receives and translates the query
		  3. Query is distributed to mixers and leaf nodes
		  4. Leaf nodes read data from Colossus
		  5. Data is aggregated through mixers
		  6. Final results are returned to the user
