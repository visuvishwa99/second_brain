# GCP BigQuery Notes
- # BigQuery Architecture
	- ## Storage and Compute
		- Storage :-> {{Colossus}} ^68086541-12d1-4412-a303-bcd7f1719174
			- Google's distributed file system for data storage
			- Columnar storage format for optimized query performance
			- Automatic management and optimization of storage
		- Compute :->  {{Dremel}} ^68086541-e87b-4296-80cd-e76d14af8886
			- Root server (Coordinates query execution)
			- Mixers (Aggregate data)
			- Leaf nodes (Read data from storage)
		- ^^MN^^ : Jupiter (gladiator) performs in Colossus with weapon called Dremel
	- ## Connecting Components
		- Jupiter:
			- High-speed network protocol connecting Dremel (compute) and Colossus (storage)
		- Borg:
			- Orchestrator managing internal communication between components
	- ## Key Architecture Concept
		- Decoupling store and compute so we pay individually
		- In RDBMS we pay store + compute together
	- ## Query Execution Flow
	  1. Query Submission: User submits an SQL query to BigQuery
	  2. Query Parsing & Optimization:
		- Root server parses SQL and converts to execution plan
		- Applies cost-based optimizations:
			- Predicate pushdown (filtering data early)
			- Join reordering
			- Pruning unused columns
			  3. Resource Allocation:
		- Borg allocates compute slots based on query complexity and priority
		  4. Distributed Execution:
		- Query broken into stages and distributed to mixers and leaf nodes
		- Leaf nodes perform parallel scans on storage
		  5. Columnar Storage Access:
		- Reads only required columns from Colossus
		- Uses Capacitor (columnar storage format)
		- Optimizes I/O by skipping unnecessary data
		- Reconstructs nested/repeated data using definition and repetition levels
		  6. Aggregation & Finalization:
		- Intermediate results aggregated by mixers
		- Final output assembled by root server
		  7. Result Delivery:
		- Final result returned to user
- # BigQuery Tables
	- ## Empty Tables
		- Tables created with defined schema but containing no data
		- Serve as structure for future data insertion or loading
	- ## Creating Tables
		- Can create from:
			- Empty table
			- Google Cloud Storage
			- Upload
			- Drive
			- Google BigTable
			- Amazon S3
			- Azure Blob storage
	- ## Partitioning Types
	  1. Integer Range
	  2. Time-unit Column (e.g., 1960-01-01, 2159-12-12)
	  3. Ingestion time
		- Note: You can't create a string as partition column in BigQuery
- # BigQuery Concepts
	- DB -> Dataset (BigQuery uses "Dataset" instead of "Database")
