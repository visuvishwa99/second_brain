- Right API
	- RDDs, DataFrames, and Datasets. DataFrames and Datasets are generally recommended over RDDs as they leverage Spark's optimizer (Catalyst) and offer better performance
- Persist()
	- Caching and persisting frequently used data in memory can greatly reduce disk I/O and recomputation overhead, leading to faster processing times. However, it's important to manage memory usage carefully.
		- Caching or persistence are optimization techniques for (iterative and interactive) Spark computations.
			- With cache(), you use only the default storage level :
			  • MEMORY_ONLY for RDD
			  • MEMORY_AND_DISK for Dataset
		- ^^Note^^ :
		- Use persist() if you want to assign a storage level other than :
		  • MEMORY_ONLY to the RDD
		  • or MEMORY_AND_DISK for Dataset
			- But Persist() We can save the intermediate results in 5 storage levels.
			  1. MEMORY_ONLY
			  2. MEMORY_AND_DISK
			  3. MEMORY_ONLY_SER (Seialized format in memory only)
			  4. MEMORY_AND_DISK_SER
			  5. DISK_ONLY
