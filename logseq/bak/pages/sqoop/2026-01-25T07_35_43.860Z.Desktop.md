- data skew in Sqoop
	- Leverage Hash-Based Splitting:
		- Sqoop offers
		  a --split-by option, but without a suitable key column, you can't directly
		  split by value. However, you can leverage hashing to create a pseudo-random
		  distribution.
		- Hash Column Values: Create a new temporary column in your source table that stores a
		  hash value of the entire text content from each row. You can use built-in
		  database functions (e.g., MD5, SHA1) for hashing.
		- Split by Hashed Column: Use the --split-by option with the newly created
		  hash column name. Sqoop will distribute the data based on the hash values,
		  aiming for a more even split across mappers.
		- Note: Hash collisions can still occur, but this approach generally provides better
		  distribution than splitting by no column at all.
